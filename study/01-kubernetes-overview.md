# 01. Kubernetes Overview

## 범위

- 쿠버네티스가 왜 필요한지
- 컨테이너 오케스트레이션이 필요한 이유
- Docker Desktop 기반 로컬 환경 준비

## 핵심 개념

- Kubernetes는 여러 컨테이너를 원하는 상태로 유지하는 오케스트레이션 도구다.
- 사람이 직접 서버를 띄우는 대신 선언형 매니페스트로 상태를 관리한다.
- 로컬 실습에서는 Docker Desktop Kubernetes가 가장 빠른 입문 환경이다.
- `kubectl`은 클러스터와 상호작용하는 기본 CLI다.

## 체크리스트

- [ ] Docker Desktop에서 Kubernetes 활성화
- [ ] `kubectl version` 확인
- [ ] `kubectl cluster-info` 확인
- [ ] `kubectl get nodes` 확인

## 명령어 메모

```bash
kubectl version
kubectl cluster-info
kubectl get nodes
```
